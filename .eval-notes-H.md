## Focus-area audit

### Tokio stdio proxy concurrency (`src/proxy.rs`)
- `PendingRequests = Arc<Mutex<HashMap<String, Pending>>>` is shared by `a2c` and `c2a` tasks.
- a2c only takes the lock to `insert(key, pend)` and immediately releases (single statement, no `.await` while held).
- c2a only takes the lock to `remove(&key)` (single statement) and on shutdown to `drain()` (still no `.await` held). The `drain()` collects into a `Vec`, drops the guard via explicit `drop(map)`, **then** does the `try_send` per pending. **No deadlock possible** — there is exactly one shared lock and no nested locking, no `.await` while holding it.
- Race on heavy load: locks serialize, but each critical section is O(1). Worst case a2c and c2a ping-pong on one mutex per frame; tokio scheduler handles that fine. Not pathological.
- ID-collision counter: a2c bumps `spans_dropped_id_collision` on `prev.is_some()`. Test `pending_insert_collision_is_observable` covers the regression path.

### Bounded channel backpressure
- `mpsc::channel::<Span>(cfg.observer_buffer)` — default 1024 (`ProxyConfig::new`).
- a2c never sends spans (only c2a does on response or orphan-drain).
- c2a uses `try_send` and bumps `spans_dropped_buffer_full` on `Err`. **Never blocks the data plane.** Matches doc claim "data-plane integrity > observation completeness."
- Drop is counted, surfaced in `ProxyMetrics`, and printed by main on exit.

### Exporter fan-out
- `replay_to` and the proxy `observe` function both iterate **all exporters** and pass each the same batch / span. **Fan-out, not round-robin.** Insertion order preserved (Vec). Verified by `replay_pushes_once_per_exporter` test and live `replay --exporter stdout` probe.

### SLO evaluation order
- `evaluate_all` does `slos.iter().map(...)` — **config order**, not alphabetical.
- TOML parser preserves array-of-tables order. Live `slo check` probe confirmed: `fast` (declared first) reported before `errs` (declared second).

### `RollingWindow` correctness (`src/slo.rs`)
- It's a **time-based** window: `cutoff = now.saturating_sub(window_nanos)`, drops front entries with `ts < cutoff`.
- Future-clock case: if a span arrives with a `ts > now`, the next `expire(later_now)` could re-evict it. Same span-clock-skew tradeoff every time-window aggregator has. Not a correctness bug; documented behavior.
- `evaluate()` (not the streaming push) recomputes filter from `spans.iter().filter(|s| s.start_unix_nanos >= cutoff && matches_tool(...))`. `now` is the max span timestamp (`latest_or_now`), so future spans get included rather than filtered. Sensible for offline mode.

### Error digest
- `error_digest(msg: &str)` takes only the **message string text** (`error.message` field) — not the structured error code or any extra fields. SHA-256 of UTF-8 bytes. Stable across exporters because all of them read the same `Span.error_message_digest` field and pass it through. **Same digest in all exporters, same digest in JSONL store.** `digest_never_leaks_plaintext` integration test confirms plaintext is never in any serialized form.

### OTLP JSON `spanId` / `traceId`
- `render_span` writes `s.span_id` and `s.trace_id` directly into the JSON. The proxy generator (`new_span_id`/`new_trace_id`) produces 16/32 lowercase hex chars (verified by trace_id.rs unit tests `span_id_is_16_hex_chars`, `trace_id_lowercase`).
- Replay-from-JSONL spans pass through as-stored — if the JSONL was produced by mcptrace, format is correct; if hand-edited, garbage in → garbage out, which is acceptable.
- OTLP `kind = 3` (`SPAN_KIND_CLIENT`) hardcoded; proxies-to-MCP-server is a CLIENT call. Test `otlp_span_kind_is_client` covers it.

### Zipkin `id` / `traceId` / `parentId`
- `id` and `traceId` passed through as-is (lowercase hex from generator).
- `parentId`: only inserted via `if let Some(p) = &s.parent_span_id { obj.insert("parentId", json!(p)) }`. **Root spans omit the field entirely** — matches Zipkin v2 spec ("absent for root spans"). Not empty string, not null.

### Exporter HTTP error handling
- Both `ZipkinExporter` and `OtlpJsonExporter` use `reqwest::blocking::Client` with `timeout(5s)`.
- On non-2xx response: returns `Error::Http(format!("zipkin returned {status}"))`. **No retry. Drop the batch, surface error.** Logged by the proxy's `observe` fn: `eprintln!("mcptrace: exporter {} error: ...")`. Documented at module top: "All built-in exporters honor a client timeout (default 5 seconds) and never retry — if an export fails, the batch is dropped and the failure is logged. This matches the observability philosophy 'data-plane integrity > observation completeness'." Consistent.
- 5xx, 4xx, network error all funnel to the same path. Replay propagates errors to caller (`replay_propagates_exporter_error` test).

### JSONL store write failure
- `JsonlStore::record` does `guard.write_all(line.as_bytes())?` and `guard.write_all(b"\n")?`. Both `?` propagate `io::Error` → `Error::Io` via `From`. The proxy `observe` catches it and logs `eprintln!("mcptrace: store.record error: {e}")` — the proxy keeps running. **Disk full / EPERM is logged but does not kill the proxy.** Correct prioritization.
- `Mutex` poisoning is handled (`map_err` → `Error::Observation`), not an `unwrap`.

### `replay --exporter stdout` end-to-end
- Live probe with a 1-span JSONL: shipped exactly once, single line of pretty JSON, exit 0, "replayed 1 spans" on stderr.

### `replay` with corrupted JSONL (one bad line)
- Live probe: `read_jsonl` **halts** at the first bad line, returns `Error::InvalidConfig("line 2: json error: ...")`. Reported by main as `Error: load spans / Caused by: invalid config: line 2: ...`. **This is HALT semantics, not skip.** Documented behavior in `read_jsonl` (returns first parse error). Acceptable — the docs frame JSONL as machine-produced; if it's corrupt the operator wants to know, not silently lose data.
- The skip-vs-halt design choice was discussed implicitly: blank lines and `#` comments ARE skipped (`read_jsonl_skips_blank_lines`, `read_jsonl_skips_comment_lines` tests). Only structurally-bad JSON halts. Reasonable middle ground.

### `stats` edge cases
- **Single span**: live probe → 1 row, p50/p95/p99 = 10, all metrics correct.
- **Zero spans (empty file)**: live probe → table header only, no rows. `compute_by_tool` returns empty `BTreeMap`, `render_table` produces a header-only table. No panic, no division by zero (`avg_ms = 0.0` early return).
- **Spans with no tool_name**: live probe with a no-tool span → grouped under `<no-tool>` row.
- **Spans with only errors**: covered in `compute_mixed_status` test; `error_rate=1.0` formats as `1.0000`.
- Percentile of empty slice → `percentile_sorted` early-returns 0.

### Duration parser units
- Supported: `ns`, `us`, `ms`, `s`, `m`, `h`, `d`. All unit-tested. No surprise: `5y`, `5w` rejected as unknown unit. No mixed compound (`1h30m` rejected — unit must come immediately after digits, single suffix). That's a deliberate keep-it-simple choice and matches doc ("regex-free loop is auditable").

## Other spot checks

- **`unsafe` audit**: `#![forbid(unsafe_code)]` in main.rs and lib.rs. Zero unsafe blocks anywhere.
- **`unwrap`/`expect` in production paths**: only inside `#[cfg(test)]`. `unreachable!()` in main.rs match arms is sound (validators already enforce variants).
- **Integer casts**: G's audit (re-verified) covered `as u64` / `as usize` boundaries — none reach truncation under realistic inputs.
- **Mutex poisoning**: every `.lock()` outside test code uses `map_err` → typed error, never `.unwrap()`.
- **`bytes vs runes`**: This is a Rust crate, no UTF-8/byte confusion. `line.as_bytes().len()` for byte counts, `String::len()` is byte length anyway.
- **CRLF on Windows**: `JsonlStore` writes `\n` explicitly (not `writeln!` which is platform-aware), test `jsonl_store_uses_lf_not_crlf` enforces it.
- **Cargo.toml dead deps**: D removed `time = "0.3"`, E updated ROUND_LOG, lockfile is consistent (build is incremental no-op).
